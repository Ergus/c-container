Detailed Description

This is a hash table with a cache for node accesses.

It uses a double linked list as an underlying container to reduce insertion and deletion costs. On the other hand there is also another underlying linked list which is modified on every access to promote and track the last accessed nodes. This method ensures that the head of the list is always the least recently used one,

On the other hand there is also a limit for the number of elements the lruTable can hold. When that number is reached, every new insertion removes the Least Recently Used one. The code also implements some optimizations to avoid calling free-malloc in that case.

Search for a node in the lruTable given a key with complexity O(1 + n/m)

When no lruTableNode with this key is present, then return NULL. This function registers all accesses to existing lruTableNode into the access cache (implemented as a double linked list).

Create a lruTableNode into the lruTable given a key and value O(1)

Inserting an element counts as an access, so the last inserted element is also the last accessed one. When the lruTable already has lruTable::maxEntries elements this operation also removes the Least Recent Used node before inserting the new one.
